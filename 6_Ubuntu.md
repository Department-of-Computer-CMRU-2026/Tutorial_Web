# Ubuntu — คู่มือสอนการใช้งานเบื้องต้นสำหรับมือใหม่

## Ubuntu คืออะไร

**Ubuntu** คือ **ระบบปฏิบัติการ (Operating System)** แบบ Linux

- **Linux** = ระบบปฏิบัติการแบบเปิดต้นฉบับ (Open Source) ใช้กันมากบนเซิร์ฟเวอร์ และบนเครื่องส่วนบุคคล
- **Ubuntu** = แดนหนึ่งของ Linux ที่ใช้งานง่าย มีทั้งแบบ **Desktop** (มีหน้าจอกราฟิก) และแบบ **Server** (ใช้คำสั่งเป็นหลัก)
- ใช้ **Terminal** (หน้าต่างคำสั่ง) เพื่อรันคำสั่ง เช่น ติดตั้งโปรแกรม, จัดการไฟล์, รันสคริปต์

เหมาะกับ: นักพัฒนา, คนที่ใช้เซิร์ฟเวอร์ Linux, คนที่อยากลองใช้ Linux บนเครื่องตัวเอง

## การได้มาและติดตั้ง Ubuntu

### วิธีใช้ Ubuntu โดยไม่ติดตั้งแทน Windows/macOS

| วิธี | ความหมายสั้นๆ |
|------|----------------|
| **WSL (Windows Subsystem for Linux)** | ใช้ Ubuntu ภายใน Windows — ติดตั้งจาก Microsoft Store หรือคำสั่ง `wsl --install` |
| **Virtual Machine (VM)** | ติดตั้ง Ubuntu ในโปรแกรมจำลองเครื่อง (เช่น VirtualBox, VMware) — ใช้ร่วมกับ Windows/macOS ได้ |
| **Live USB** | บูตจาก USB แล้วลองใช้ Ubuntu โดยไม่ติดตั้ง (หรือติดตั้งจากนั้นได้) |

### ติดตั้ง Ubuntu แทนหรือคู่กับ Windows (บนเครื่องจริง)

- ดาวน์โหลดไฟล์ ISO จาก [ubuntu.com/download](https://ubuntu.com/download/)
- สร้าง USB Bootable (ด้วย Rufus, Balena Etcher ฯลฯ) แล้วบูตเครื่องจาก USB
- เลือก **Install Ubuntu** แล้วทำตามขั้นตอน (สามารถเลือกติดตั้งคู่กับ Windows ได้)

### ติดตั้ง WSL บน Windows (แนะนำถ้าใช้ Windows)

1. เปิด PowerShell หรือ Command Prompt **แบบ Administrator**
2. รันคำสั่ง:

```bash
wsl --install
```

3. รีสตาร์ทเครื่องเมื่อติดตั้งเสร็จ
4. เปิด **Ubuntu** จากเมนู Start — จะได้ Terminal แบบ Ubuntu

- ถ้าต้องการเฉพาะ Ubuntu: `wsl --install -d Ubuntu`
- ตรวจสอบเวอร์ชัน: เปิด Ubuntu แล้วรัน `lsb_release -a`

## ส่วนติดต่อผู้ใช้ (Desktop) พื้นฐาน

ถ้าคุณใช้ **Ubuntu Desktop** (มีหน้าจอกราฟิก):

| ส่วน | ความหมายสั้นๆ |
|------|----------------|
| **Desktop** | พื้นที่หลัก มีไอคอนและวอลเปเปอร์ |
| **Top bar** | แถบด้านบน มีเมนู, เวลา, การตั้งค่า, ปิดเครื่อง |
| **Side bar / Dock** | แถบด้านซ้าย มีไอคอนโปรแกรมที่ใช้บ่อย |
| **Files (ไฟล์)** | โปรแกรมเปิดโฟลเดอร์ — คลิกไอคอนโฟลเดอร์หรือเปิดจากเมนู |
| **Terminal** | เปิดจากเมนู Applications หรือกด **Ctrl+Alt+T** |

- ถ้าใช้ **Ubuntu Server** หรือ **WSL** จะไม่มี Desktop — ใช้ผ่าน Terminal อย่างเดียว

## Terminal — เปิดและใช้คำสั่ง

**Terminal** คือหน้าต่างที่ใช้พิมพ์ **คำสั่ง (Command)** เพื่อให้ระบบทำงาน เช่น ดูไฟล์, สร้างโฟลเดอร์, ติดตั้งโปรแกรม

### เปิด Terminal

- **Ubuntu Desktop:** กด **Ctrl+Alt+T** หรือเปิดจากเมนู Applications → Terminal
- **WSL:** เปิด **Ubuntu** จากเมนู Start (Windows)
- **macOS/Linux อื่น:** เปิดแอป Terminal หรือ Konsole ฯลฯ

### พื้นฐานการพิมพ์คำสั่ง

- พิมพ์คำสั่งแล้วกด **Enter** ระบบจะรันคำสั่งนั้น
- เส้นทางโฟลเดอร์และชื่อไฟล์ **ต้องตรงตัวอักษร** (รวมตัวใหญ่-เล็ก บน Linux)
- **พาธ (Path):**
  - **`/`** = รากของระบบ (root)
  - **`~`** = โฟลเดอร์บ้านของผู้ใช้ปัจจุบัน (เช่น `/home/username`)
  - **`.`** = โฟลเดอร์ปัจจุบัน
  - **`..`** = โฟลเดอร์ระดับบน

---

## การเลื่อนโฟลเดอร์และดูไฟล์

### คำสั่งที่ใช้บ่อย

| คำสั่ง | ความหมาย | ตัวอย่าง |
|--------|-----------|----------|
| **`pwd`** | แสดงโฟลเดอร์ปัจจุบัน (Print Working Directory) | `pwd` |
| **`ls`** | แสดงรายการไฟล์และโฟลเดอร์ในโฟลเดอร์ปัจจุบัน | `ls` |
| **`ls -la`** | แสดงรายการแบบละเอียด รวมไฟล์ซ่อน (ขึ้นต้นด้วย `.`) | `ls -la` |
| **`cd ชื่อโฟลเดอร์`** | เข้าไปในโฟลเดอร์นั้น | `cd Documents` |
| **`cd ..`** | กลับไปโฟลเดอร์ระดับบน | `cd ..` |
| **`cd ~`** | ไปที่โฟลเดอร์บ้าน | `cd ~` |
| **`cd /path/to/folder`** | ไปที่เส้นทางแบบเต็ม | `cd /home/user/Downloads` |

### ตัวอย่างการใช้งาน

```bash
pwd                    # ดูว่าอยู่โฟลเดอร์ไหน
ls                     # ดูว่ามีไฟล์/โฟลเดอร์อะไรบ้าง
cd Documents           # เข้าโฟลเดอร์ Documents
cd ..                  # กลับขึ้นหนึ่งระดับ
cd ~/Downloads         # ไปที่โฟลเดอร์ Downloads ในโฟลเดอร์บ้าน
```

## การสร้าง/ลบ/คัดลอก/ย้ายไฟล์และโฟลเดอร์

### สร้างโฟลเดอร์

```bash
mkdir ชื่อโฟลเดอร์
mkdir project/src      # สร้าง project แล้วสร้าง src ข้างใน (ต้องมี -p ถ้า project ยังไม่มี)
mkdir -p project/src   # สร้างได้แม้ project ยังไม่มี
```

### สร้างไฟล์ว่าง

```bash
touch ชื่อไฟล์
touch readme.txt
```

### คัดลอก (Copy)

```bash
cp ไฟล์ต้นทาง ไฟล์ปลายทาง
cp file.txt file_backup.txt
cp -r โฟลเดอร์ต้นทาง โฟลเดอร์ปลายทาง   # -r = ทั้งโฟลเดอร์ (recursive)
cp -r myfolder myfolder_backup
```

### ย้ายหรือเปลี่ยนชื่อ (Move / Rename)

```bash
mv ต้นทาง ปลายทาง
mv oldname.txt newname.txt    # เปลี่ยนชื่อ
mv file.txt ~/Documents/      # ย้ายไปโฟลเดอร์อื่น
```

### ลบ (Remove)

```bash
rm ชื่อไฟล์                  # ลบไฟล์
rm file.txt
rm -r ชื่อโฟลเดอร์           # ลบโฟลเดอร์และของข้างใน (ระวัง: ลบแล้วกู้ยาก)
rm -rf โฟลเดอร์             # ลบโดยไม่ถามยืนยัน — ใช้ด้วยความระมัดระวัง
```

- **คำเตือน:** `rm -rf` ลบแล้วกู้คืนไม่ได้ ตรวจเส้นทางให้ดีก่อนรัน

### ดูเนื้อหาไฟล์

```bash
cat ชื่อไฟล์                 # แสดงเนื้อหาทั้งไฟล์
cat readme.txt
less ชื่อไฟล์                # แสดงทีละหน้ากด Space เลื่อน กด q ออก
head -n 10 ไฟล์              # แสดง 10 บรรทัดแรก
tail -n 10 ไฟล์              # แสดง 10 บรรทัดท้าย
```

## การติดตั้งโปรแกรม (apt)

Ubuntu ใช้ระบบ **apt** (Advanced Package Tool) สำหรับติดตั้ง/อัปเดต/ลบโปรแกรมจาก repository

### อัปเดตรายการแพ็กเกจ

```bash
sudo apt update
```

- ควรรันก่อน `apt install` เพื่อให้ได้รายการแพ็กเกจล่าสุด

### ติดตั้งโปรแกรม

```bash
sudo apt install ชื่อแพ็กเกจ
sudo apt install git
sudo apt install nodejs npm
sudo apt install php php-mysql
```

- **sudo** = รันคำสั่งในฐานะผู้ดูแลระบบ (จะถามรหัสผ่าน)
- ชื่อแพ็กเกจมักเป็นตัวพิมพ์เล็ก ไม่มีช่องว่าง

### ค้นหาแพ็กเกจ

```bash
apt search คำค้น
apt search editor
```

### ลบโปรแกรม

```bash
sudo apt remove ชื่อแพ็กเกจ
sudo apt purge ชื่อแพ็กเกจ     # ลบรวมการตั้งค่า
```

### อัปเกรดโปรแกรมที่ติดตั้งแล้ว

```bash
sudo apt update
sudo apt upgrade
```

## การแก้ไขไฟล์ข้อความ (nano)

**nano** คือโปรแกรมแก้ไขข้อความใน Terminal ใช้ง่ายสำหรับมือใหม่

### เปิดไฟล์

```bash
nano ชื่อไฟล์
nano readme.txt
```

- ถ้าไฟล์ไม่มี nano จะสร้างไฟล์ใหม่เมื่อบันทึก

### การใช้งานใน nano

| การกระทำ | วิธีทำ |
|----------|--------|
| **เลื่อน cursor** | ลูกศร |
| **บันทึก** | **Ctrl+O** แล้วกด Enter |
| **ออก** | **Ctrl+X** (จะถามบันทึกหรือไม่ถ้ามีการแก้ไข) |
| **ค้นหา** | **Ctrl+W** แล้วพิมพ์คำที่ต้องการหา |
| **ตัดบรรทัด** | **Ctrl+K** |
| **วาง** | **Ctrl+U** |

- ด้านล่างของ nano จะแสดงคำสั่งลัด (^ = Ctrl)

### ตัวแก้ไขตัวอื่น

- **vim** / **vi** — ทรงพลังแต่ต้องเรียนคำสั่ง (กด `i` เพื่อพิมพ์ กด `Esc` แล้วพิมพ์ `:wq` เพื่อบันทึกและออก)
- **gedit** (Desktop) — เปิดจาก Terminal: `gedit ชื่อไฟล์` ถ้ามี GUI

## ผู้ใช้และสิทธิ์ (sudo)

- บน Linux แต่ละคนเป็น **ผู้ใช้ (user)** — โฟลเดอร์บ้านอยู่ที่ `/home/ชื่อผู้ใช้`
- คำสั่งบางอย่าง (เช่น ติดตั้งโปรแกรม, แก้ไฟล์ระบบ) ต้องใช้สิทธิ์ **ผู้ดูแลระบบ (root)**
- **sudo** = รันคำสั่งด้วยสิทธิ์สูง (จะถามรหัสผ่านของผู้ใช้ปัจจุบัน)

### ตัวอย่าง

```bash
sudo apt update          # อัปเดตรายการแพ็กเกจ (ต้องใช้ sudo)
sudo nano /etc/hostname  # แก้ไฟล์ระบบ (ต้องใช้ sudo)
```

### ข้อควรระวัง

- ใช้ `sudo` เฉพาะเมื่อจำเป็น — คำสั่งที่รันด้วย sudo มีผลกับทั้งระบบ
- อย่ารันคำสั่งที่คนอื่นให้มาโดยไม่เข้าใจ โดยเฉพาะที่ขึ้นต้นด้วย `sudo`

## คำสั่งและเทคนิคที่ใช้บ่อย

### สรุปคำสั่งพื้นฐาน

| การกระทำ | คำสั่ง |
|----------|--------|
| ดูโฟลเดอร์ปัจจุบัน | `pwd` |
| ดูรายการไฟล์ | `ls` หรือ `ls -la` |
| เข้าโฟลเดอร์ | `cd ชื่อโฟลเดอร์` หรือ `cd ..` |
| สร้างโฟลเดอร์ | `mkdir ชื่อโฟลเดอร์` |
| สร้างไฟล์ว่าง | `touch ชื่อไฟล์` |
| คัดลอก | `cp ต้นทาง ปลายทาง` (โฟลเดอร์ใช้ `cp -r`) |
| ย้าย/เปลี่ยนชื่อ | `mv ต้นทาง ปลายทาง` |
| ลบไฟล์ | `rm ชื่อไฟล์` (โฟลเดอร์ใช้ `rm -r`) |
| ดูเนื้อหาไฟล์ | `cat ชื่อไฟล์` หรือ `less ชื่อไฟล์` |
| แก้ไขข้อความ | `nano ชื่อไฟล์` |
| ติดตั้งโปรแกรม | `sudo apt update` แล้ว `sudo apt install ชื่อแพ็กเกจ` |

### เทคนิคใน Terminal

- **ล้างหน้าจอ:** `clear` หรือ **Ctrl+L**
- **หยุดคำสั่งที่กำลังรัน:** **Ctrl+C**
- **เติมคำสั่งอัตโนมัติ (Tab):** พิมพ์ส่วนหนึ่งของชื่อไฟล์/โฟลเดอร์แล้วกด **Tab** — จะเติมให้หรือแสดงตัวเลือก
- **ประวัติคำสั่ง:** กด **ลูกศรขึ้น** เพื่อเรียกคำสั่งที่เคยใช้
- **ลากไฟล์/โฟลเดอร์ลง Terminal:** บาง Terminal จะใส่เส้นทางให้อัตโนมัติ

## สรุปแนวทางสำหรับมือใหม่

1. **ได้ Ubuntu มา:** ติดตั้ง WSL บน Windows (`wsl --install`) หรือใช้ VM / ติดตั้งบนเครื่องจริง ตามที่สะดวก
2. **เปิด Terminal:** กด Ctrl+Alt+T (Desktop) หรือเปิด Ubuntu จาก Start (WSL)
3. **เลื่อนโฟลเดอร์:** ใช้ `pwd`, `ls`, `cd` — จำ `cd ..` (กลับขึ้น) และ `cd ~` (ไปโฟลเดอร์บ้าน)
4. **จัดการไฟล์:** ใช้ `mkdir`, `touch`, `cp`, `mv`, `rm` — ระวัง `rm -r` และ `rm -rf` ลบแล้วกู้ยาก
5. **ดูเนื้อหาไฟล์:** ใช้ `cat` หรือ `less` — แก้ไขด้วย `nano ชื่อไฟล์`
6. **ติดตั้งโปรแกรม:** รัน `sudo apt update` แล้ว `sudo apt install ชื่อแพ็กเกจ` (เช่น git, nodejs)
7. **ใช้ Tab และลูกศรขึ้น:** Tab เติมชื่อไฟล์/โฟลเดอร์ ลูกศรขึ้นเรียกคำสั่งเก่า
8. **มี error:** อ่านข้อความที่ Terminal แสดง มักบอกสาเหตุ (เช่น No such file, Permission denied)

ถ้าทำตามขั้นตอนนี้และลองใช้คำสั่ง `pwd`, `ls`, `cd`, `mkdir`, `nano` ในโฟลเดอร์บ้าน จะเริ่มคุ้นกับ Ubuntu และ Terminal ได้เร็ว
