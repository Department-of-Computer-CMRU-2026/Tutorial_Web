# คู่มือการใช้งาน Docker (แบบเข้าใจง่าย)

## Docker คืออะไร (แบบสั้นมาก)

คิดง่ายๆ ว่า Docker คือโปรแกรมที่ทำให้คุณ **รันแอปอื่นๆ ใน “กล่อง” แยกจากเครื่อง**  
กล่องนั้นมีทุกอย่างที่แอปต้องการ (เช่น โปรแกรมย่อย, ไฟล์ตั้งค่า) เลยรันได้เหมือนกันทุกเครื่องที่ติดตั้ง Docker

- **Image** = แบบ/ต้นแบบของกล่อง (ยังไม่รัน)
- **Container** = กล่องที่รันอยู่จริง (เปิดแล้ว)

คุณไม่ต้องจำรายละเอียดเทคนิค แค่รู้ว่า: **ดึง Image มา → สั่งรัน → ได้ Container** แล้วเข้าไปใช้หรือเปิดเว็บได้ตามที่โปรเจกต์บอก

---

## ติดตั้งและเปิดใช้

### 1. ติดตั้ง Docker

1. ไปที่ [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop)
2. ดาวน์โหลด **Docker Desktop** ของระบบคุณ (Windows / Mac / Linux)
3. ติดตั้งเหมือนโปรแกรมทั่วไป (Next / ตกลง ไปเรื่อยๆ)

### 2. เปิด Docker ทุกครั้งก่อนใช้

- เปิดโปรแกรม **Docker Desktop** ให้รันอยู่ (มีไอคอนอยู่ที่ tray หรือ menu bar)
- ถ้าไม่เปิด Docker Desktop คำสั่ง `docker` ใน Terminal จะใช้ไม่ได้

### 3. ตรวจสอบว่าพร้อมใช้

เปิด **Terminal** (หรือ Command Prompt / PowerShell) แล้วพิมพ์:

```bash
docker --version
```

ถ้าแสดงเลขเวอร์ชัน (เช่น `Docker version 24.0...`) แสดงว่าพร้อมใช้

ลองรันคำสั่งทดสอบ:

```bash
docker run hello-world
```

ถ้ามีข้อความต้อนรับภาษาอังกฤษขึ้นมา แสดงว่าติดตั้งและใช้งานได้ปกติ

## เปิด-ปิด-ลบ โปรแกรมที่รันอยู่

### ดูว่ามีอะไรรันอยู่

ใน Terminal พิมพ์:

```bash
docker ps
```

จะเห็นรายการ “กล่อง” (container) ที่กำลังรันอยู่ มีชื่อหรือรหัสสั้นๆ ใช้สำหรับสั่งเปิด-ปิด-ลบ

### หยุด (ปิดชั่วคราว)

```bash
docker stop web
```

แทนที่ `web` ด้วยชื่อที่คุณตั้งไว้ (จาก `--name`) หรือชื่อที่แสดงใน `docker ps`

### เปิดใหม่ (หลังหยุดแล้ว)

```bash
docker start web
```

### ลบ (ให้หายจากรายการ)

ถ้าหยุดแล้วและอยากลบออกจากเครื่อง:

```bash
docker rm web
```

ถ้ายังรันอยู่และอยากบังคับลบ:

```bash
docker rm -f web
```

## ถ้ามีไฟล์ docker-compose ในโปรเจกต์

บางโปรเจกต์จะมีไฟล์ชื่อ **docker-compose.yml** หมายความว่าเขาตั้งค่า “รันหลายอย่างพร้อมกัน” ไว้แล้ว (เช่น แอป + ฐานข้อมูล) คุณแค่รันคำสั่งเดียว

### วิธีใช้

1. เปิด **Terminal**
2. ไปที่โฟลเดอร์ที่มีไฟล์ **docker-compose.yml** (มักเป็นโฟลเดอร์หลักของโปรเจกต์):

   ```bash
   cd /path/to/project
   ```

   (แทนที่ `/path/to/project` ด้วยเส้นทางจริง เช่น `cd ~/Desktop/my-app`)

3. รันคำสั่ง:

   ```bash
   docker compose up -d
   
   docker compose exec app php artisan migrate
   ```

   - `up` = เปิดทุกอย่างที่เขากำหนดไว้
   - `-d` = รันไว้หลังพื้นหลัง (ปิด Terminal ไปก็ยังรันอยู่)

4. เปิดเว็บหรือใช้แอปตามที่โปรเจกต์บอก (เช่น เปิด `http://localhost:3000`)

### หยุดทั้งหมด

เมื่อใช้เสร็จ อยู่ที่โฟลเดอร์เดิมแล้วรัน:

```bash
docker compose down
```

ทุกอย่างที่เปิดด้วย `docker compose up` จะหยุดและถูกลบออกจากรายการ (ข้อมูลใน volume อาจยังอยู่ ขึ้นกับที่เขาตั้งค่า)

### ถ้าเขาบอกให้ “build ใหม่”

บางทีเขาอาจบอกให้รัน:

```bash
docker compose up -d --build
```

แปลว่า “อัปเดตแล้วให้ build ใหม่แล้วค่อยเปิด” — ใช้เมื่อมีแก้ Dockerfile หรือตั้งค่าแล้วเขาบอกให้ build ใหม่

## สรุปคำสั่งที่ใช้บ่อย

| อยากทำอะไร | คำสั่ง |
|------------|--------|
| ดูว่า Docker พร้อมไหม | `docker --version` |
| ดูว่ามีอะไรรันอยู่ | `docker ps` |
| หยุด | `docker stop ชื่อ` |
| เปิดใหม่ | `docker start ชื่อ` |
| ลบ | `docker rm ชื่อ` หรือ `docker rm -f ชื่อ` (ถ้ายังรันอยู่) |
| เปิดทั้งโปรเจกต์ (มี docker-compose) | `cd โฟลเดอร์โปรเจกต์` แล้ว `docker compose up -d` |
| ปิดทั้งโปรเจกต์ | `docker compose down` |

---

## สรุปสั้นๆ

- **ติดตั้ง Docker Desktop** แล้ว **เปิดโปรแกรมก่อนใช้** ทุกครั้ง
- **เปิดเว็บ** ตามที่เขาบอก (เช่น `http://localhost:8080`)
- ถ้ามี **docker-compose.yml** ให้เข้าโฟลเดอร์นั้นแล้วรัน **`docker compose up -d`** เพื่อเปิด แล้วรัน **`docker compose down`** เมื่อจะปิด
- อยากหยุดหรือลบอะไร ใช้ **`docker stop ชื่อ`** หรือ **`docker rm ชื่อ`** ตามที่สรุปไว้ด้านบน
