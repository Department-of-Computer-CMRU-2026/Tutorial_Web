## FileZilla คืออะไร

**FileZilla** คือ **โปรแกรม FTP/SFTP Client** สำหรับส่งและรับไฟล์ระหว่างเครื่องคุณกับเซิร์ฟเวอร์

- **FTP** = โปรโตคอลสำหรับส่งไฟล์ (File Transfer Protocol) — ใช้กันมานาน
- **SFTP** = ส่งไฟล์ผ่านช่องทางที่เข้ารหัส (Secure) — แนะนำใช้แทน FTP เมื่อเซิร์ฟเวอร์รองรับ
- ใช้ **เชื่อมต่อ** กับเซิร์ฟเวอร์ (เช่น โฮสต์เว็บ) แล้ว **อัปโหลด/ดาวน์โหลด** ไฟล์และโฟลเดอร์ได้
- มีทั้ง **FileZilla Client** (ฟรี, ใช้บนเครื่องเรา) และ **FileZilla Server** (ติดตั้งบนเซิร์ฟเวอร์) — คู่มือนี้เน้น **Client**

เหมาะกับ: คนอัปโหลดเว็บ, นักพัฒนา, คนที่ต้องส่ง/รับไฟล์กับเซิร์ฟเวอร์ผ่าน FTP หรือ SFTP

## การติดตั้ง FileZilla

### ดาวน์โหลดและติดตั้ง

- ไปที่ [filezilla-project.org/download.php](https://filezilla-project.org/download.php?type=client)
- เลือก **FileZilla Client** แล้วดาวน์โหลดตามระบบปฏิบัติการ
- **Windows:** ติดตั้งจากไฟล์ `.exe` (ระวังตัวติดตั้งอาจมีซอฟต์แวร์เสริม — เลือกเฉพาะ FileZilla ถ้าไม่ต้องการ)
- **macOS:** เปิดไฟล์ `.dmg` แล้วลาก FileZilla ไปที่ Applications
- **Linux:** ใช้ package manager เช่น `sudo apt install filezilla` (Ubuntu/Debian) หรือดาวน์โหลดจากเว็บ

### เปิดโปรแกรมครั้งแรก

- เปิด FileZilla จะเห็นแถบด้านบนสำหรับใส่ Host, Username, Password, Port และปุ่ม **Quickconnect**
- ด้านซ้าย = ไฟล์บนเครื่องเรา (Local) ด้านขวา = ไฟล์บนเซิร์ฟเวอร์ (Remote) หลังเชื่อมต่อแล้ว

## ส่วนติดต่อผู้ใช้ (UI) พื้นฐาน

เมื่อเปิด FileZilla จะเห็นพื้นที่หลักประมาณนี้:

| ส่วน | ความหมายสั้นๆ |
|------|----------------|
| **แถบด้านบน (Toolbar)** | Quick Connect (Host, User, Password, Port), ปุ่มเชื่อมต่อ/ตัดการเชื่อมต่อ, Site Manager |
| **Local site (ซ้าย)** | โฟลเดอร์และไฟล์บน **เครื่องของคุณ** — ใช้เลือกโฟลเดอร์ต้นทางเวลาอัปโหลด/ดาวน์โหลด |
| **Remote site (ขวา)** | โฟลเดอร์และไฟล์บน **เซิร์ฟเวอร์** — ใช้เลือกโฟลเดอร์ปลายทาง |
| **แถบล่าง (Transfer queue)** | แสดงรายการไฟล์ที่กำลังส่ง/รับ และผลลัพธ์ (สำเร็จ/ล้มเหลว) |
| **Message log (ล่าง)** | แสดงข้อความจากโปรแกรมและเซิร์ฟเวอร์ (คำสั่ง FTP, error ฯลฯ) |

### การเลื่อนโฟลเดอร์

- **Local (ซ้าย):** ดับเบิลคลิกโฟลเดอร์เพื่อเข้าไป หรือใช้ลูกศรกลับ/ไปข้างหน้า
- **Remote (ขวา):** ทำแบบเดียวกัน — หลังเชื่อมต่อแล้วจะเห็นโฟลเดอร์บนเซิร์ฟเวอร์

## การเชื่อมต่อเซิร์ฟเวอร์ (FTP/SFTP)

### ข้อมูลที่ต้องมีจากผู้ให้บริการโฮสต์

ก่อนเชื่อมต่อต้องรู้ค่าจากผู้ให้บริการ (โฮสต์) มักมีใน Control Panel (cPanel, Plesk ฯลฯ) หรืออีเมลยืนยันการสมัคร:

| ค่า | ความหมาย | ตัวอย่าง |
|----|-----------|-----------|
| **Host / Server** | ที่อยู่เซิร์ฟเวอร์ FTP | `ftp.example.com` หรือ `123.45.67.89` |
| **Username** | ชื่อผู้ใช้ FTP | `youruser` หรือ `youruser@example.com` |
| **Password** | รหัสผ่าน FTP | (ที่โฮสต์ให้หรือที่คุณตั้ง) |
| **Port** | พอร์ต (ถ้าไม่ใส่จะใช้ค่าเริ่มต้น) | FTP: `21`, SFTP: `22` |

### เชื่อมต่อแบบ Quick Connect

1. ที่แถบด้านบน กรอก **Host**, **Username**, **Password**
2. ถ้าใช้ **SFTP** ให้ใส่ **Port** เป็น `22` (ถ้าเป็น FTP ปกติ Port มักเป็น `21` หรือเว้นไว้)
3. กดปุ่ม **Quickconnect**
4. ครั้งแรกที่เชื่อมต่อ SFTP อาจมีหน้าต่างถามเรื่อง **Unknown host key** — ถ้าเชื่อถือเซิร์ฟเวอร์ให้กด **OK** (หรือติ๊กจำ key แล้ว OK)
5. เมื่อเชื่อมต่อสำเร็จ ด้าน **Remote site (ขวา)** จะแสดงโฟลเดอร์บนเซิร์ฟเวอร์

### ประเภทโปรโตคอล (Protocol)

- **FTP** — พอร์ต 21 (ค่าเริ่มต้น) ไม่เข้ารหัส — ใช้เมื่อโฮสต์ให้เฉพาะ FTP
- **SFTP (SSH File Transfer Protocol)** — พอร์ต 22 — **แนะนำ** เพราะเข้ารหัส
- ในช่อง Host บางโฮสต์ให้ใส่เป็น `sftp://ftp.example.com` เพื่อบังคับใช้ SFTP หรือเลือกโปรโตคอลใน Site Manager

### เมื่อเชื่อมต่อไม่ได้

- ตรวจ **Host, Username, Password, Port** ให้ตรงกับที่โฮสต์ให้
- ตรวจว่าไฟร์วอลล์หรือเราเตอร์ไม่บล็อกพอร์ต 21/22
- ดูข้อความใน **Message log** ด้านล่าง มักบอกสาเหตุ (เช่น Login incorrect, Connection refused)

## การเรียกดูและเลื่อนโฟลเดอร์

### Local (เครื่องเรา)

- ใช้รายการโฟลเดอร์ด้านซ้าย **ดับเบิลคลิก** โฟลเดอร์เพื่อเข้าไป
- ปุ่ม **ลูกศรขึ้น** ใช้กลับไปโฟลเดอร์ระดับบน
- สามารถพิมพ์เส้นทางโฟลเดอร์ในช่องด้านบน (Local path) ได้

### Remote (เซิร์ฟเวอร์)

- หลังเชื่อมต่อแล้ว ด้านขวาจะแสดงโฟลเดอร์บนเซิร์ฟเวอร์
- **ดับเบิลคลิก** โฟลเดอร์เพื่อเข้าไป (เช่น เข้า `public_html` หรือ `www` เพื่ออัปโหลดไฟล์เว็บ)
- ปุ่ม **ลูกศรขึ้น** ใช้กลับไปโฟลเดอร์ระดับบน
- เส้นทางปัจจุบันจะแสดงที่แถบด้านบน (Current server path)

### โฟลเดอร์ที่ใช้กับเว็บไซต์โดยทั่วไป

- โฮสต์ส่วนใหญ่มีโฟลเดอร์ชื่อ **`public_html`** หรือ **`www`** หรือ **`httpdocs`** — ไฟล์ที่อัปโหลดไปที่นี่จะถูกแสดงเมื่อเปิดโดเมนในเบราว์เซอร์
- ตรวจจากคู่มือโฮสต์หรือ Control Panel ว่า "Document Root" อยู่ที่โฟลเดอร์ไหน

## การอัปโหลดและดาวน์โหลดไฟล์

### อัปโหลด (เครื่องเรา → เซิร์ฟเวอร์)

1. ด้าน **Local (ซ้าย)** ไปที่โฟลเดอร์ที่มีไฟล์ที่ต้องการส่ง
2. ด้าน **Remote (ขวา)** ไปที่โฟลเดอร์ปลายทางบนเซิร์ฟเวอร์ (เช่น `public_html`)
3. เลือกไฟล์หรือโฟลเดอร์ที่ต้องการอัปโหลด (คลิกหรือ Ctrl+Click / Cmd+Click เพื่อเลือกหลายรายการ)
4. **ลากไปวาง** ที่พื้นที่ Remote หรือ **คลิกขวา** → **Upload** หรือกดปุ่มลูกศร **→** (ไปทางขวา)
5. รายการจะไปอยู่ใน **Transfer queue** ด้านล่าง และจะอัปโหลดตามลำดับ

### ดาวน์โหลด (เซิร์ฟเวอร์ → เครื่องเรา)

1. ด้าน **Remote (ขวา)** เลือกไฟล์หรือโฟลเดอร์ที่ต้องการดาวน์โหลด
2. ด้าน **Local (ซ้าย)** ไปที่โฟลเดอร์ปลายทางบนเครื่องเรา
3. **ลากไปวาง** ที่พื้นที่ Local หรือ **คลิกขวา** → **Download** หรือกดปุ่มลูกศร **←** (ไปทางซ้าย)
4. รายการจะไปอยู่ใน Transfer queue และดาวน์โหลดตามลำดับ

### การเลือกหลายไฟล์

- **Ctrl+Click** (Windows/Linux) หรือ **Cmd+Click** (macOS) = เลือกทีละไฟล์
- **Shift+Click** = เลือกเป็นช่วงจากรายการที่เลือกอยู่ถึงรายการที่คลิก
- **Ctrl+A** / **Cmd+A** = เลือกทั้งหมดในรายการปัจจุบัน

### การเขียนทับไฟล์ (Overwrite)

- ถ้ามีไฟล์ชื่อเดียวกันอยู่แล้ว FileZilla จะถามว่า **Overwrite**, **Overwrite if source newer**, **Skip** ฯลฯ
- สามารถตั้งค่าเริ่มต้นได้ที่ **Edit** → **Settings** → **Transfers** → **File exists action**

## การจัดการไฟล์และโฟลเดอร์บนเซิร์ฟเวอร์

### สร้างโฟลเดอร์ใหม่ (Remote)

1. ไปที่โฟลเดอร์บน Remote ที่ต้องการสร้างโฟลเดอร์ย่อย
2. **คลิกขวา** ในพื้นที่รายการไฟล์ Remote → **Create directory** หรือกดปุ่มสร้างโฟลเดอร์
3. ใส่ชื่อโฟลเดอร์ แล้วกด **OK**

### เปลี่ยนชื่อ (Rename)

- **คลิกขวา** ที่ไฟล์หรือโฟลเดอร์ (Local หรือ Remote) → **Rename**
- ใส่ชื่อใหม่ แล้วกด **OK**

### ลบ (Delete)

- เลือกไฟล์หรือโฟลเดอร์ แล้ว **คลิกขวา** → **Delete** หรือกด **Delete**
- **ระวัง:** การลบบน Remote จะลบของบนเซิร์ฟเวอร์จริง — ตรวจให้ดีก่อนลบ

### คัดลอก URL (สำหรับไฟล์บนเซิร์ฟเวอร์)

- **คลิกขวา** ที่ไฟล์บน Remote → **Copy URL(s) to clipboard** — ใช้แชร์ลิงก์หรือใส่ในเบราว์เซอร์ได้

## การแก้ไขสิทธิ์ (Permissions)

บางโฮสต์ต้องการให้ตั้งสิทธิ์ไฟล์/โฟลเดอร์ (เช่น 755 สำหรับโฟลเดอร์, 644 สำหรับไฟล์) เพื่อให้เว็บทำงานถูกต้อง

### วิธีเปลี่ยนสิทธิ์ (CHMOD)

1. **คลิกขวา** ที่ไฟล์หรือโฟลเดอร์บน **Remote** → **File permissions** หรือ **Permissions**
2. ใส่ค่าเป็นตัวเลข (Numeric) หรือติ๊กช่อง Read/Write/Execute ตามต้องการ
3. เลือก **Recurse into subdirectories** ถ้าต้องการให้ใช้กับโฟลเดอร์ย่อยและไฟล์ข้างใน
4. กด **OK**

### ค่าที่ใช้บ่อย

| ค่า | ความหมาย | ใช้กับ |
|-----|-----------|--------|
| **755** | เจ้าของอ่าน+เขียน+รัน, คนอื่นอ่าน+รัน | โฟลเดอร์ (มักใช้กับโฟลเดอร์บนเว็บ) |
| **644** | เจ้าของอ่าน+เขียน, คนอื่นอ่านอย่างเดียว | ไฟล์ (เช่น .html, .css, .js, .jpg) |
| **775** | เจ้าของและกลุ่มอ่าน+เขียน+รัน, คนอื่นอ่าน+รัน | โฟลเดอร์ที่ต้องให้สคริปต์เขียนได้ |
| **600** | เจ้าของอ่าน+เขียนอย่างเดียว | ไฟล์สำคัญ (เช่น config ที่มีรหัสผ่าน) |

- ค่าที่โฮสต์แนะนำอาจต่างกัน — ดูจากคู่มือโฮสต์หรือ Support

## Site Manager — บันทึกการเชื่อมต่อ

ถ้าเชื่อมต่อเซิร์ฟเวอร์เดิมบ่อย ควรบันทึกไว้ใน **Site Manager** จะได้ไม่ต้องพิมพ์ Host/User/Password ทุกครั้ง

### เปิด Site Manager

- กดปุ่ม **Site Manager** (ไอคอนโฟลเดอร์กับเครื่องหมายบวก) ที่แถบด้านบน หรือ **File** → **Site Manager** หรือ **Ctrl+S** (Windows/Linux) / **Cmd+S** (macOS)

### เพิ่ม Site ใหม่

1. กด **New Site** แล้วตั้งชื่อ (เช่น "My Web Host")
2. กรอกข้อมูล:
   - **Protocol:** FTP หรือ SFTP (แนะนำ SFTP ถ้าโฮสต์รองรับ)
   - **Host:** ที่อยู่เซิร์ฟเวอร์
   - **Port:** เว้นว่างได้ (ใช้ค่าเริ่มต้น) หรือใส่ 21 (FTP) / 22 (SFTP)
   - **Logon Type:** Normal (ใส่ User + Password) หรือ Ask for password (ให้ถามรหัสทุกครั้ง)
   - **User:** ชื่อผู้ใช้
   - **Password:** รหัสผ่าน (หรือเว้นไว้ถ้าเลือก Ask for password)
3. กด **Connect** เพื่อทดสอบ หรือ **OK** เพื่อบันทึก

### เชื่อมต่อจาก Site Manager

- เปิด Site Manager → เลือก Site ที่บันทึกไว้ → กด **Connect**
- หรือเลือก Site จาก dropdown ด้านบน (ถ้ามี) แล้วกด Quickconnect

### แก้ไข / ลบ Site

- เปิด Site Manager → เลือก Site → กด **Edit** หรือ **Delete**

| อัปโหลดที่เลือก | — | ลากไปขวา หรือปุ่ม → |
| ดาวน์โหลดที่เลือก | — | ลากไปซ้าย หรือปุ่ม ← |
| เลือกทั้งหมด | Ctrl+A | Cmd+A |
| สร้างโฟลเดอร์ | — | คลิกขวา → Create directory |

### เทคนิคที่ใช้บ่อย

- **ลากวาง:** ลากไฟล์จาก Local ไปวางที่ Remote = อัปโหลด ลากจาก Remote มาวางที่ Local = ดาวน์โหลด
- **รีเฟรช:** ถ้าเห็นรายการไม่ตรงกับบนเซิร์ฟเวอร์ ให้คลิกที่พื้นที่ Remote แล้วกด **F5**
- **ดู Message log:** ถ้าเชื่อมต่อหรือส่งไฟล์ไม่สำเร็จ ดูข้อความที่แถบล่าง (Message log) มักบอกสาเหตุ
- **ยกเลิกการส่ง:** ใน Transfer queue คลิกขวารายการที่กำลังส่ง → **Cancel**

## ข้อควรระวังและความปลอดภัย

1. **ใช้ SFTP แทน FTP เมื่อทำได้** — FTP ส่งรหัสผ่านไม่เข้ารหัส SFTP เข้ารหัสทั้งหมด
2. **อย่าเก็บรหัสผ่านใน Site Manager บนเครื่องสาธารณะ** — ใช้ Logon Type แบบ "Ask for password" หรือไม่บันทึกรหัส
3. **ตรวจก่อนลบ** — การลบไฟล์/โฟลเดอร์บน Remote จะลบของบนเซิร์ฟเวอร์จริง
4. **สำรองข้อมูล** — ก่อนแก้หรือลบไฟล์สำคัญบนเซิร์ฟเวอร์ ควรดาวน์โหลดเก็บไว้ก่อน
5. **ดาวน์โหลดจากแหล่งที่เชื่อถือได้** — ใช้เฉพาะจาก [filezilla-project.org](https://filezilla-project.org) เพื่อหลีกเลี่ยงตัวติดตั้งที่มีมัลแวร์

## สรุปแนวทางสำหรับมือใหม่

1. **ติดตั้ง FileZilla Client** จากเว็บอย่างเป็นทางการ แล้วเปิดโปรแกรม
2. **เตรียมข้อมูลจากโฮสต์** — Host, Username, Password, Port (FTP 21 / SFTP 22)
3. **เชื่อมต่อ:** กรอกที่แถบ Quick Connect แล้วกด **Quickconnect** หรือบันทึกใน **Site Manager** (Ctrl+S / Cmd+S) แล้วกด Connect
4. **ไปที่โฟลเดอร์ปลายทางบนเซิร์ฟเวอร์** — มักเป็น `public_html` หรือ `www` สำหรับไฟล์เว็บ
5. **อัปโหลด:** เลือกไฟล์ด้าน Local (ซ้าย) แล้วลากไปวางที่ Remote (ขวา) หรือคลิกขวา → Upload
6. **ดาวน์โหลด:** เลือกไฟล์ด้าน Remote (ขวา) แล้วลากไปวางที่ Local (ซ้าย) หรือคลิกขวา → Download
7. **สร้างโฟลเดอร์/เปลี่ยนชื่อ/ลบ:** ใช้คลิกขวาบน Remote หรือ Local ตามต้องการ
8. **ตั้งสิทธิ์ (Permissions):** คลิกขวาที่ไฟล์/โฟลเดอร์บน Remote → File permissions แล้วใส่ค่า (เช่น 755, 644)
9. **มีปัญหา:** ดู Message log ด้านล่าง และตรวจ Host/User/Password/Port ให้ตรงกับโฮสต์

ถ้าทำตามขั้นตอนนี้และลองเชื่อมต่อกับเซิร์ฟเวอร์จริงสักหนึ่งตัว แล้วลองอัปโหลด/ดาวน์โหลดไฟล์ จะเริ่มคุ้นกับ FileZilla ได้เร็ว
